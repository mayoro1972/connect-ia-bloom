Guide Administrateur n8n V3 - TransferAI Africa

Version de reference : 5 mai 2026

Objet du document
Ce guide explique comment exploiter, maintenir et superviser le workflow n8n V3 de reponse IA Chatwoot pour TransferAI Africa.

1. Resume executif

La V3 remplace la reponse statique de la V2 par une reponse generee par OpenAI, tout en gardant une architecture simple et un rollback rapide vers la V2 en cas de besoin.

2. Workflow actif recommande

  • Actif : ACTIVE - TransferAI Chatwoot Auto Reply V3 (OpenAI)
  • Fallback : ACTIVE - TransferAI Chatwoot Auto Reply V2 (Fallback statique)
  • Archives : workflows V1 desactives

3. Webhook de production V3

https://n8n-pxlk.srv1480638.hstgr.cloud/webhook/chatwoot-inbound-v3

4. Architecture V3

  • Chatwoot Webhook
  • Filter + Build reply (Code)
  • OpenAI Chat Completions
  • Send reply to Chatwoot

5. Ce que fait le nœud Code

  • verifie l'evenement Chatwoot
  • ignore les messages sortants
  • ignore les messages vides
  • extrait conversation_id
  • prepare le texte entrant et la reponse a envoyer

6. Configuration HTTP Chatwoot

  • Method : POST
  • URL : https://app.chatwoot.com/api/v1/accounts/163278/conversations/{{$json.conversation_id}}/messages
  • Header : api_access_token
  • Header : Content-Type = application/json

7. Configuration OpenAI

Le header Authorization doit utiliser la forme :

Bearer sk-...

La cle ne doit pas etre commit dans le repo. Utiliser des placeholders dans les exports de workflow.

8. Test de fonctionnement

  1. Envoyer un message depuis le widget Chatwoot.
  2. Verifier qu'une execution apparait dans n8n.
  3. Verifier que tous les nœuds V3 passent au vert.
  4. Verifier que la reponse IA apparait dans la conversation Chatwoot.

9. Rollback rapide

Si la V3 pose probleme, remettre temporairement l'URL du webhook Chatwoot sur le path V2 :

https://n8n-pxlk.srv1480638.hstgr.cloud/webhook/chatwoot-inbound-v2

10. Erreurs deja rencontrées

  • boucle infinie sur messages sortants
  • champ URL invalide a cause du prefixe =
  • acces refuse aux variables d'environnement
  • header OpenAI sans prefixe Bearer
  • instabilite du noeud If

11. Regles de stabilite

  • ne pas reintroduire le nœud If a la place du nœud Code
  • ne pas commit de vrai token dans le repo
  • conserver une V2 active comme filet de securite pendant la phase de stabilisation
  • changer un seul parametre a la fois en cas de debug

12. Prochaine etape

La suite recommandee est la V4 Memoire, qui reutilise l'historique Chatwoot pour eviter de reposer les memes questions au prospect.
