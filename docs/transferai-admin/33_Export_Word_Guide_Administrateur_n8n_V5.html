Objet du document
Ce guide explique comment exploiter, superviser et maintenir la V5 du workflow Chatwoot + n8n de TransferAI. La V5 ajoute la qualification structuree, l’ecriture dans les custom attributes Chatwoot, l’application automatique de labels et la capture de contact renforcee.

1. Resume executif

La V5 conserve les acquis des V3 et V4 : reponse IA contextualisee et memoire conversationnelle. Elle ajoute une deuxieme brique OpenAI dediee a la qualification commerciale, puis ecrit les resultats dans Chatwoot pour rendre les conversations directement actionnables par les agents.

2. Workflow actif recommande

Type Nom recommande Role
Production ACTIVE - TransferAI Chatwoot Auto Reply V5 (Qualification) Workflow principal en charge de la reponse, qualification, labels et custom attributes.
Rollback ACTIVE - TransferAI Chatwoot Auto Reply V4 (Memoire) Filet de securite si la V5 doit etre contournee rapidement.
Reference historique ACTIVE - TransferAI Chatwoot Auto Reply V3 (OpenAI) Base stable historique, conservee comme reference documentaire.

5. Ce que la V5 ajoute

  • Extraction JSON stricte du besoin commercial.
  • Calcul automatique de labels de traitement.
  • Ecriture des champs de qualification dans Chatwoot.
  • Capture de contact renforcee avec Option B.
  • Preparation de la suite commerciale vers demo ou handoff humain.

6. Option B de collecte de contact

Strategie retenue
TransferAI utilise une double securite : un Pre Chat Form Chatwoot pour capter le nom et l’email en amont, et une logique de workflow qui redemande l’email seulement s’il manque au moment opportun.
  • Pre Chat Form :
    • fullName obligatoire
    • emailAddress obligatoire
    • phoneNumber optionnel
  • Workflow V5 :
    • ne re-demande jamais l’email s’il est deja connu
    • demande l’email s’il manque avant une proposition de demo
    • mappe lead_email et lead_phone dans Chatwoot

7. Labels et custom attributes attendus

Labels

Les labels suivants doivent exister dans Chatwoot avec l’orthographe exacte :

formation, audit-ia, veille, automatisation, catalogue, certification, webinar, partenariat, emploi, support, lead-chaud, infos-manquantes, handoff-humain, demo-proposee.

Custom attributes

Au minimum :

company_name, sector, need_type, intent, lead_seriousness, internal_summary, next_best_action, is_b2b, demo_proposed, human_handoff, lead_email, lead_phone.

8. Configuration OpenAI et Chatwoot

La V5 utilise :

  • 4 noeuds avec token Chatwoot dans les headers
  • 2 noeuds avec cle OpenAI prefixed par Bearer
Regle de securite
Les vraies cles et tokens ne doivent jamais etre commit dans le repo. Les exports versionnes doivent conserver les placeholders.

9. Validation fonctionnelle observee

  • les 12 noeuds ont ete testes pas a pas
  • la reponse IA est bien renvoyee dans Chatwoot
  • les labels sont bien appliques
  • les custom attributes sont bien ecrits
  • lead_email et lead_phone sont bien pris en compte
  • si l’email est deja connu, il n’est pas redemande
  • si l’email est absent, le bot peut le demander avant une demo

10. Test de fonctionnement recommande

  1. Envoyer un message de qualification riche depuis le widget.
  2. Verifier qu’une execution apparait dans n8n.
  3. Verifier que les 12 noeuds passent au vert.
  4. Verifier la reponse IA dans Chatwoot.
  5. Verifier la presence des labels et des custom attributes.
  6. Tester un cas sans email explicite pour confirmer la logique Option B.

11. Rollback

Si la V5 pose probleme, remettre temporairement le webhook Chatwoot sur la V4 :

https://n8n-pxlk.srv1480638.hstgr.cloud/webhook/chatwoot-inbound-v4

12. Regles de stabilite

  • ne pas activer de mock data sur le webhook en production
  • ne pas modifier plusieurs noeuds a la fois sans re-test
  • garder la V4 active 24 a 48h apres bascule
  • surveiller les erreurs OpenAI et Chatwoot dans l’onglet Executions
  • regenerer un token s’il a ete expose dans une capture

13. Prochaine etape

La suite logique est la V5.5 Calendly conditionnel : meme socle que la V5, mais avec une proposition de demo plus stricte, seulement si l’email est connu et le lead suffisamment mature.
