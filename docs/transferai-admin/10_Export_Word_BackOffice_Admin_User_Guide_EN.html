TransferAI Africa - BackOffice Administrator Guide

Exportable Word version - updated on 1 May 2026

Purpose: provide administrators with a practical and current guide to operate the public BackOffice, the supporting backend, and the connected workflows without breaking live production.
Operational scope: this guide reflects the current production stack, including Analytics, Audit Prospects, Resources, AI Drafts, AI Partners, AI Newsletter, Video Capsules, WhatsApp, AI Jobs, Webinars, AI Live Formats, and the How to Use section.

1. Mission of the primary administrator

The primary administrator keeps the platform coherent, commercially credible, operationally reliable, and traceable. This role supervises content governance, access control, live workflows, and final validation before publication.

2. How to access the system

  • Public BackOffice: https://www.transferai.ci/back-office
  • Direct tab example: https://www.transferai.ci/back-office?tab=whatsapp
  • Local development access: http://127.0.0.1:<port>/back-office
  • Supabase: Table Editor, Edge Functions, Secrets, SQL Editor
  • Twilio: inbound WhatsApp logs and sender controls
  • Resend: outbound email logs and domain configuration

3. Administrator token and backend access

The public BackOffice requires a valid administrator token. The backend validates this against CONTENT_ADMIN_TOKEN through the content-admin edge function.

Good practice:

  • do not share the token in unsecured channels
  • rotate it if exposure is suspected
  • retest the BackOffice immediately after any token rotation

8. Core workflows

WhatsApp

  1. visitor clicks the site WhatsApp CTA
  2. message is sent to +2250716573990
  3. Twilio receives the message
  4. Twilio calls the Supabase webhook
  5. the message is stored in whatsapp_inbound_messages
  6. internal email notifications are sent
  7. the message becomes visible in BackOffice > WhatsApp

Newsletter

The operating model remains: AI prepares, human validates, system sends.

  • Wednesday: draft generation
  • Thursday: approval reminder if still pending
  • Friday: approved issue can be sent

Partners

  1. a partner request is submitted
  2. the request is reviewed
  3. a recommended path is assigned
  4. follow-up is prepared and sent

9. Recommended operating routine

Daily

  • check contact and audit requests
  • check incoming WhatsApp messages
  • verify the BackOffice loads properly
  • verify no critical public-page issue is visible

Weekly

  • review AI drafts
  • review the newsletter issue and send a test
  • check partner requests and follow-up status
  • check the featured video capsule

Monthly

  • review top pages and CTAs
  • review operational copy on high-impact pages
  • review secrets and access handling
  • update priorities and documentation

10. Common issues and first checks

  • If the BackOffice does not load: verify the admin token, Supabase variables, and the content-admin function.
  • If WhatsApp messages do not appear: check Twilio logs, twilio-whatsapp-webhook recent invocations, and whatsapp_inbound_messages.
  • If WhatsApp notification emails are not seen: check whatsapp_email_notification_logs, Resend logs, and inbox / spam / promotions folders.
  • If the public page looks outdated: verify that the change reached main, production deployment, and browser cache refresh.

11. Final recommendation

The BackOffice should be treated as a live operations cockpit, not just a publishing interface. The safest operating model remains clear ownership, human review for sensitive outputs, disciplined deployment checks, and systematic logging of critical workflows.
